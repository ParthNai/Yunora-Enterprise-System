====================================================
YUNORA WEBSITE ↔ ADMIN PANEL ↔ DATABASE CONNECTION SYSTEM
=========================================================

This is a mandatory requirement.

The Website, Admin Panel, Database, Storage System, APIs, CRM, Warranty System, Payment System, and Dealer Portal must all be connected through a single centralized backend architecture.

Nothing should be hardcoded.

Everything must be dynamic.

====================================================
CENTRAL DATABASE ARCHITECTURE
=============================

Use a centralized database.

Recommended:

• PostgreSQL (Preferred)
or
• MySQL

All data must be stored in database.

Examples:

• Products
• Categories
• Collections
• Orders
• Customers
• Dealers
• Reviews
• Blogs
• Offers
• Banners
• Homepage Content
• Warranty Records
• Contact Forms
• Support Tickets
• Coupons
• Analytics

No content should be manually edited inside website code.

====================================================
DYNAMIC WEBSITE CONNECTION
==========================

Whenever Admin updates anything:

• Product
• Price
• Banner
• Offer
• Collection
• Blog
• Contact Information
• Homepage Content

Changes must automatically appear on website.

No redeployment required.

No coding required.

Instant update system required.

====================================================
SUPER ADMIN REQUIREMENT
=======================

The Admin Panel should control 100% of the website.

If a new product, banner, offer, blog, category, collection, dealer, review, warranty, Instagram post, contact detail, or homepage content is added from Admin Panel, it must automatically appear on the website without touching the code.

The website and admin panel must behave as one connected ecosystem.

====================================================
FINAL REQUIREMENT
=================

Build the architecture in such a way that:

• Future Mobile App can connect easily.
• Future ERP can connect easily.
• Future CRM can connect easily.
• Future AI modules can connect easily.

The entire YUNORA platform should operate from a single database and a single admin control center while updating the website in real time.
